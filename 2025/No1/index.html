技术

这篇文章描述了 Agent 系统开发过程中，涉及到的工程复杂度，包含可运行性（Run）、可复现性（Reproduce）和可进化性（Evolve）。其中可运行性由 Agent 框架帮开发人员解决了，当我们通过几行代码调用 LLM 就能跑起来一个 Agent 后，我们可能会产生 Agent 系统很容易开发的错觉。但其实后两者才是我们要去面对和解决的问题。作者给出了他对 Agent 的理解：框架幻觉 –> 场景拼接 –> 系统设计 –> 工程落地 –> 智能演化。

谷歌新书 Agentic Design Pattern


一个中文翻译的项目

七猫对 AIOps 对通用 Agent 的探索，使用了 ReAct 设计思想，即引导模型将复杂问题进行拆分，一步一步地进行推理（Reasoning）和行动（Action），同时还引入了观察（Observation）环节，在每次执行之后，都会先观察当前现状，然后再进行下一步的推理。
该探索实践公开的仓库地址：https://codeup.aliyun.com/qimao/public/aiops/aiops，在 Agent 实现上使用了字节开源的 Eino 开发框架。
